# 动态报告规划

报告结构必须由用户的分析目标和数据可回答的问题决定，不使用固定模板硬套。

## 报告规划流程

1. 识别用户决策问题：用户要据此决定什么？
2. 判断需求类型：增长规划、归因诊断、分群对比、活动评估、漏斗、cohort、风控或实验。
3. 根据数据字段判断可回答的问题和不可回答的问题。
4. 为每个核心问题生成一个分析章节。
5. 每个章节绑定一张证据表、一张图和一个行动建议。
6. 把所有章节组织成有叙事顺序的报告。

## 分析章节对象

每个正文分析章节都按以下对象规划：

## 章节顺序建议

### 增长规划类

1. 当前经营基线和目标设定：趋势图。
2. 增长来自哪里：SKU/地区/渠道贡献图。
3. 哪些因素拖累增长质量：退款、低转化、低客单风险图。
4. 资源应该投向哪里：分层矩阵或 Top-N 图。
5. 下期行动路线：优先级表。

### 归因诊断类

1. 指标是否真的变化：趋势图和基线。
2. 变化发生在哪里：贡献度拆解图。
3. 是否由个别异常驱动：异常点图。
4. 最可能原因和待验证假设：证据表。
5. 修复动作和监控指标。

### 分群对比类

1. 分群表现总览：排序条形图。
2. 高价值群体画像：Top-N 或矩阵。
3. 高风险群体画像：风险排序图。
4. 资源分配建议：优先级矩阵。

### 活动/优惠评估类

1. 主指标表现：销售额、转化或订单提升。
2. 护栏指标表现：退款率、客单价、质量风险。
3. 分群差异：不同券型/人群/渠道对比。
4. 保留、调整或停止建议。

## 每节一图规则

- 一个核心分析章节原则上只放一张最能回答问题的图。
- 图后必须紧跟解释：图说明了什么、为什么重要、该做什么。
- 如果一个问题需要多个图，拆成多个章节。
- 附录可以放补充表，不要把补充图塞进正文稀释重点。

## 报告骨架

动态报告仍保留少量稳定外壳：

```markdown
## 摘要

用 3-6 条回答用户的核心问题，每条包含数字和动作。

## 数据范围与口径

说明输入、时间范围、核心指标契约和限制。

## 分析章节 1：由用户问题生成

结论 + 图 + 证据 + 解释 + 行动。

## 分析章节 2：由用户问题生成

结论 + 图 + 证据 + 解释 + 行动。

## 行动计划

按优先级列出动作、目标指标、负责人/节奏建议和风险。

## 附录：产物索引

列出 Base 表、仪表盘组件、图表图片和关键字段。
```

不要固定使用“关键发现 1/2/3”这种泛化标题。章节标题应是业务问题或结论，例如“Q3 先修复退款，再恢复增长”。
