# 分析链路规划

本文件用于把用户需求、计算过程、飞书 Base、仪表盘和飞书文档报告连成一条可追溯链路。做分析前先规划，发布前逐项核对。

## 链路契约

每个核心结论必须能沿以下路径回溯：

`用户目标 -> 决策问题 -> 计算步骤 -> 过程/结果表 -> 图表数据表 -> 图表注册 -> 仪表盘组件 -> 报告章节 -> 文档块`

使用稳定 ID 串联产物：

| 类型 | ID 示例 | 用途 |
| --- | --- | --- |
| 决策问题 | `Q01` | 用户要回答的业务问题 |
| 计算步骤 | `S01` | 可复现的原子计算动作 |
| 图表 | `CH01` | Base 仪表盘组件和本地图表图片的统一标识 |
| 报告章节 | `SEC01` | 正文分析章节 |
| 文档块 | `RB01_TEXT`, `RB01_CHART` | 飞书文档分段发布顺序 |

## 按需求类型选择计算链路

### 增长规划

- 基线步骤：时间趋势、最近完整周期、同比/环比和波动区间。
- 增量步骤：品类、SKU、地区、渠道或人群贡献拆解。
- 护栏步骤：退款率、取消率、低毛利、低复购或履约风险。
- 行动步骤：机会分层、优先级排序和下季度目标拆解。

### 归因诊断

- 变化确认：目标指标是否真实变化，是否受缺失或周期影响。
- 贡献拆解：哪个维度贡献了主要变化。
- 异常识别：异常时间点、异常 SKU、异常渠道。
- 假设输出：区分已证实事实和待验证原因。

### 活动/优惠评估

- 主指标：销售额、订单数、转化率、客单价或拉新。
- 护栏指标：退款率、毛利、复购、履约异常。
- 分群差异：券型、人群、渠道、商品层级。
- 决策建议：保留、放大、调整或停止。

## 发布前核对

- 每个 `SECxx` 至少关联一个 `Sxx`、一个证据表和一个 `CHxx`，除非该章节是数据范围或附录。
- 每个 `CHxx` 必须出现在 `97_图表注册表`、`dashboard.blocks` 和 `doc.blocks`。
- 每个 `Sxx` 的输出表必须写入 Base，且在 `00_任务规划` 中记录状态。
- 每个正文图表图片必须与对应章节相邻发布，优先使用 manifest 的 `doc.blocks`。
- 不允许只有报告文字而没有 Base 证据，也不允许有仪表盘图表但报告没有解释。
- 不把全量明细当作默认计算过程表。原始文件是事实源；Base 默认保存可复盘的口径、过程摘要和结果表。
