# 报告写作指南

## 核心原则

报告不是模板填空，而是围绕用户决策问题组织证据。先判断用户想据此做什么决定，再决定报告章节、分析方法和图表。

每个正文分析章节必须包含：

1. 一个业务问题或结论式标题。
2. 一句话短答案。
3. 一张最能支撑该章节的图。
4. 图后的证据解释，包含关键数字和对比基线。
5. 置信度和限制。
6. 可执行建议。
7. 可回溯引用：`章节ID`、证据 Base 表、图表 ID、仪表盘组件和图片说明。

## 决策简报格式

每个章节按以下顺序写：

```markdown
## Q3 是否应该先扩量？

结论：不建议直接激进扩量，Q3 应先把退款率作为增长护栏。

![图：退款率趋势](...)

证据：
- Q2 退款率为 xx%，高于 Q1 的 xx%。
- 高退款 SKU 贡献了 xx% 的退款订单。

解释：
这说明销售增长可能被售后损耗抵消，单看 GMV 会高估经营质量。

建议：
把退款率纳入投放准入，先治理高销售高退款 SKU，再扩大预算。

置信度：中。限制：缺少退款原因和履约成本字段，暂不能判断具体根因。
```

## 摘要写法

摘要只回答用户的核心问题，不复述所有分析过程。

- 用 3-6 条要点。
- 每条包含结论、关键数字和动作。
- 优先写“该怎么做”，再写“为什么”。
- 如果结论置信度不高，直接说明还缺什么数据。

## 数据范围与口径

报告前部保留简短的数据说明：

- 输入文件和工作表。
- 时间范围和样本量。
- 核心指标契约：指标名、粒度、分子、分母、过滤条件。
- 数据质量限制。
- 可追溯产物：Base 表、仪表盘组件、图表图片名称。

## 正文章节写法

- 标题使用业务问题或结论，不用“分析一/分析二”。
- 每节只解决一个问题。
- 每节原则上只放一张图；如果需要两张图，拆成两节。
- 图必须紧跟解释，不要把所有图堆到报告末尾。
- 解释中引用确切的 Base 表和仪表盘组件名称。
- 章节中应出现对应的证据关系，例如 `章节 SEC02；图表 CH02；Base 表：90_图表数据_SKU表现；仪表盘组件：高退款 SKU 风险排行`。
- 结论必须区分事实、推断和建议。

## 行动计划写法

最后用表格沉淀行动计划：

| 优先级 | 行动 | 目标指标 | 证据来源 | 风险/依赖 |
| --- | --- | --- | --- | --- |
| 高 | 治理高退款 SKU | 退款率下降 x 个百分点 | `90_图表数据_SKU表现` | 需要退款原因字段 |

## 飞书 Markdown 注意事项

- 调用 `docs +create --title` 时，正文开头不要重复写文档标题作为一级标题。
- 适度使用标题、表格、列表和 callout，保持可读性。
- 图表图片应插入在解释该图的段落之前或之后，但必须与该章节相邻。
- 本地图表图片使用 `docs +media-insert` 插入。
