# 报告写作指南

## 推荐结构

除非用户指定其他格式，否则使用以下结构：

```markdown
## 摘要

用 3-6 条要点回答用户的核心问题。每条要点包含结论、关键数字、业务含义。

## 数据范围与方法

- 输入文件：
- 分析周期：
- 关键字段：
- 数据质量限制：
- 可追溯产物：Base 表和仪表盘名称

## 分析过程

用表格列出步骤、计算方法、产出表、用途。

## 关键发现

按重要性组织，每个发现包含：
1. 结论
2. 证据数字
3. 来源 Base 表
4. 对应仪表盘组件或图表图片
5. 业务解释

## 行动建议

给出可执行动作、优先级、预期收益、风险或依赖。

## 附录：产物索引

列出 Base 表、仪表盘组件、图表图片、关键字段说明。
```

## 写作标准

- 每个结论都要回扣用户的分析目标。
- 优先使用具体数字，不使用含糊形容词。
- 解释模式为什么重要，而不仅说明发生了什么变化。
- 引用精确的产物名称，例如 `Base 表：90_图表数据_渠道转化` 和 `仪表盘组件：渠道转化漏斗`。
- 区分事实、推断和建议。
- 保留缺失数据、重复数据、异常值、时间窗口不一致或假设条件带来的限制。
- 不把所有计算都堆进正文；过程追溯细节放在 Base 中，并在报告里引用。

## 飞书 Markdown 注意事项

- 调用 `docs +create --title` 时，正文开头不要重复写文档标题作为一级标题。
- 适度使用标题、表格、列表和 callout，保持可读性。
- 图表图片应插入在解释该图的段落之后。
- 本地图表图片使用 `docs +media-insert` 插入。
