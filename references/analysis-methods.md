# 分析方法选择

先从用户的决策问题出发，而不是从数据字段出发。分析开始前判断：如果结果是 A 或 B，用户会改变什么行动？

## 需求类型到分析方法

| 用户问题类型 | 典型表达 | 优先方法 | 关键产物 |
| --- | --- | --- | --- |
| 增长规划 | 下季度规划、经营建议、目标怎么定 | 时间趋势 + 结构拆解 + 分层建议 | 目标基准、增长抓手、风险约束 |
| 归因诊断 | 为什么下降、为什么波动、哪里异常 | 趋势对比 + 贡献度拆解 + 异常检测 | 主要驱动项、异常区间、可验证假设 |
| 分群对比 | 哪些 SKU/地区/渠道好，怎么分层 | 分组聚合 + Top/Bottom + 风险收益矩阵 | 分层名单、优先级、资源分配建议 |
| 优惠/活动效果 | 优惠券效果、活动有没有用 | 对照组思路 + 主指标/护栏指标 + 分群比较 | 活动收益、客单价、退款/质量护栏 |
| 漏斗转化 | 哪里流失、转化率如何提升 | 漏斗分析 + 阶段 drop-off + 分群漏斗 | 最大流失点、修复动作 |
| 留存/cohort | 新老用户表现、复购、留存 | cohort 分析 + cohort size + 留存/收入矩阵 | cohort 曲线、改善或恶化趋势 |
| 质量风控 | 退款、异常、履约、低质订单 | 异常检测 + 分层风险 + 根因假设 | 高风险对象、治理优先级 |
| 实验评估 | A/B、策略是否有效 | 假设检验 + 效应量 + 置信区间 | 上线/迭代/停止决策 |

## 方法使用要点

### 时间趋势

用于判断指标随时间变化。输出趋势方向、变化幅度、异常点和可能结构性断点。比较不同长度周期时，先归一到日均、周均或同口径周期。

### 结构拆解

用于回答“变化由谁贡献”。按 SKU、地区、渠道、券型、客户类型等维度拆分，同时保留总量、占比和变化量。

### 分群对比

用于找增长机会或风险对象。不要只看平均值；同时看规模、转化/销售、质量风险和稳定性。

### 漏斗分析

用于多步骤转化。必须定义阶段、时间窗口和同一实体口径。输出各阶段转化率和最大流失点。

### cohort 分析

用于随时间变化的用户/订单行为。必须定义 cohort 起点、事件和观察窗口。输出 cohort size，避免小样本过度解释。

### 假设检验

用于比较两个或多个群体差异是否可靠。输出效应量和置信区间，不只给显著性。多重比较时降低结论置信度。

### 异常检测

用于找异常记录、异常时间点或高风险对象。可用 IQR、标准差、业务规则或模型方法。异常只代表“需要调查”，不要直接当成原因。

## 指标契约

每个核心指标在报告中首次出现时，至少说明：

- 业务问题：该指标回答什么问题。
- 粒度：订单、用户、商品、地区、天、周、月等。
- 分子/分母：尤其是比率指标。
- 过滤条件：是否排除退款、测试数据、异常记录。
- 时间窗口和时区。
- 数据来源和限制。

不满足以下条件时，不要把指标当作稳定结论：

- 前后周期的分子、分母和过滤条件一致。
- 对比周期长度或粒度一致，或已归一化。
- 数据源和字段含义清楚。
- 百分比有对应分母。

## 分析陷阱

- 不要平均百分比；应汇总分子和分母后重新计算。
- 不要把相关性写成因果，除非有实验或明确识别策略。
- 不要只报告看起来好的切片；说明探索过的主要维度。
- 不要用过小样本做强结论。
- 不要把累计图当作近期趋势证据。
- 不要忽略季节性、节假日、活动、履约变化等混杂因素。
